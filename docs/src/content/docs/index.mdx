```typescript title="app.ts"
import { Queue, Worker } from 'bunqueue/client';

const queue = new Queue('emails', { embedded: true });

await queue.add('welcome', {
  to: 'user@example.com',
  subject: 'Welcome aboard!',
  template: 'onboarding',
});

const worker = new Worker('emails', async (job) => {
  await sendEmail(job.data);
  await job.updateProgress(100);
  return { sent: true };
}, { embedded: true, concurrency: 5 });

worker.on('completed', (job, result) => {
  console.log(`Done: ${job.id}`, result);
});

worker.on('failed', (job, error) => {
  console.error(`Failed: ${job.id}`, error.message);
});
```

    <p>Stall detection, dead letter queues, retries with backoff, rate limiting, S3 backups. All built-in.</p>

  <div class="hp-bento-card hp-bento-wide">
    <h3>Native MCP Server for AI Agents</h3>
    <p>73 MCP tools, 5 resources, 3 prompts. AI agents can schedule tasks, manage pipelines, monitor queues, and control everything via natural language. Works with Claude, Cursor, Windsurf, and any MCP client. HTTP handlers let agents process jobs without deploying workers.</p>
  </div>
  <div class="hp-bento-card hp-bento-wide">
    <h3>Drop-in BullMQ Replacement</h3>
    <p>Familiar Queue and Worker API. Migrate from BullMQ in minutes, not days. Same patterns, same mental model, 32x better performance.</p>
  </div>
  <div class="hp-bento-card">
    <h3>Full TypeScript</h3>
    <p>End-to-end type safety with comprehensive generics. Your IDE autocompletes everything.</p>
  </div>
  <div class="hp-bento-card">
    <h3>Cron &amp; Scheduling</h3>
    <p>Cron expressions, delayed jobs, repeatable tasks. Schedule anything with timezone support.</p>
  </div>
  <div class="hp-bento-card">
    <h3>Simple Mode</h3>
    <p>Queue + Worker in one object. Routes, middleware, cron, circuit breaker, batch processing, deduplication &mdash; zero boilerplate.</p>

  <div class="hp-bento-card hp-bento-wide">

Saga compensation, conditional branching, parallel steps, retry with backoff, nested sub-workflows, signal timeouts, and typed observability events. No Temporal, no Inngest, no cloud service &mdash; just <code>bunqueue/workflow</code>.</p>
