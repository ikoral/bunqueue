bunqueue includes an **MCP (Model Context Protocol)** server that gives AI assistants **complete control** over your job queues. 66 tools, 5 resources, zero configuration.

## What is MCP?

MCP is an open protocol that enables AI assistants to interact with external tools and services. With bunqueue's MCP server, you can:

- Add, pull, ack, and fail jobs using natural language
- Monitor queue health, memory, and Prometheus metrics
- Configure cron jobs, rate limits, and concurrency
- Debug failed jobs in the DLQ
- Manage workers, webhooks, and locks
- Full pull/ack/fail cycle for job consumption

<Aside type="tip">
MCP works with Claude Desktop, Claude Code (CLI), Cursor, Windsurf, and any MCP-compatible client.
</Aside>

## Installation

The MCP server is included with bunqueue. No additional installation required.

```bash
bun add bunqueue
```

### Remote Server (TCP Mode)

Connect to a running bunqueue server instead of embedded SQLite:

```json
{
  "mcpServers": {
    "bunqueue": {
      "command": "bunx",
      "args": ["bunqueue-mcp"],
      "env": {
        "BUNQUEUE_MODE": "tcp",
        "BUNQUEUE_HOST": "localhost",
        "BUNQUEUE_PORT": "6789",
        "BUNQUEUE_TOKEN": "your-auth-token"
      }
    }
  }
}
```

## Connection Modes

|                  | Embedded (default)           | TCP (remote)                     |
| ---------------- | ---------------------------- | -------------------------------- |
| **How it works** | Direct SQLite access         | Connects to bunqueue server      |
| **Setup**        | Zero config                  | Set `BUNQUEUE_MODE=tcp`          |
| **Best for**     | Local dev, single-machine    | Production, shared queues        |
| **Data**         | `./data/bunq.db`             | Remote server handles storage    |

## Data Persistence

The MCP server uses SQLite for persistence. Default location:

```
./data/bunq.db
```

Configure with environment variable:

```json
{
  "mcpServers": {
    "bunqueue": {
      "command": "bunx",
      "args": ["bunqueue-mcp"],
      "env": {
        "DATA_PATH": "/var/lib/queue.db"
      }
    }
  }
}
```

## Processing Jobs

<Aside type="caution">
The MCP server **manages** queues. You need a separate Worker to **process** jobs.
</Aside>

Run a worker alongside the MCP server:

```typescript
// worker.ts
import { Worker } from 'bunqueue/client';

new Worker('emails', async (job) => {
  const { to, subject, body } = job.data;

  await job.updateProgress(50, 'Sending email...');
  await sendEmail(to, subject, body);
  await job.log('Email sent successfully');

  return { sent: true, timestamp: Date.now() };
}, { embedded: true });

console.log('Worker started, processing emails queue...');
```

```bash
bun run worker.ts
```

## Troubleshooting

### MCP server not connecting

```bash
# Verify installation
bunx bunqueue-mcp --help

# Check if Bun is in PATH
which bun

# Test manually
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | bunx bunqueue-mcp
```

### Jobs stuck in waiting

Make sure a Worker is running:

```bash
bun run worker.ts
```

### Data not persisting

Check file permissions:

```bash
ls -la ./data/
# Should show bunq.db with write permissions
```

### Claude not seeing tools

1. Restart Claude Desktop/Code
2. Check config file syntax (valid JSON)
3. Verify path to `bunqueue-mcp` is correct
4. Run `bunx bunqueue-mcp` manually to check for errors
